Creating Texture Inspector Tools

Creating a tool for the texture inspector is very easy. You just need to write a short .js file and then host it somewhere online; I have been using GH Pages for this purpose. Here is a sample tool file: Paintbrush.js. You can load this tool into the editor by clicking the "add tool" button, and then pasting in that URL and clicking add. The important things to note:

  1. the .js file must be structured as _TOOL_DATA_ = {...your tool data...}.
  2. You must define a name (string), type (class), and icon (base 64 encoded SVG icon.
  3. There are two optional properties. usesWindow if you want a GUI window and is3D if you want to use post processes.
  4. The class must have a constructor, which should save the function getParameters to a member variable. getParameters returns the tools API in the form of an object.
  5. The class must have a setup() function which is called when the tool is selected in the toolbar.
  6. The class must have a cleanup() function which is called when the tool is deselected.
  7. You must call updateTexture() after you make any changes to the texture. (which is a member of getParameters())
  8. 2D tools can use the canvas API to paint on the texture.
  9. 3D tools must create post processes or otherwise modify scene3D.
